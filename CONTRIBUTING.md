# Como contribuir

Obrigado por contribuir com os encontros mensais do LABHDUFBA!

## Fluxo de trabalho

1. **Crie um branch** a partir de `main` com um nome descritivo:
   ```bash
   git checkout -b encontro/2026-07-03-texto
   ```

2. **Faça suas alterações** — adicione materiais, preencha o `README.md` do encontro, atualize o cronograma etc.

3. **Abra um Pull Request** para `main` com uma descrição breve do que foi adicionado ou alterado.

4. **Marque um revisor** — escolha pelo menos um(a) membro(a) do laboratório para revisar antes de mesclar.

5. Após a aprovação, o revisor ou o autor pode fazer o merge.

## Convenções

- Nome de branch: `encontro/YYYY-MM-DD-tipo` para materiais de encontros; `fix/descricao` para correções gerais
- Pastas de encontro sempre no padrão `YYYY-MM-DD-tipo/` dentro de `encontros/`
- Cada pasta de encontro deve ter um `README.md` com os campos: Tema, Responsável, Resumo, Materiais e Notas
